What is Pylint?

Pylint is a source-code, bug and quality checker for the Python programming language. In the professional world, Pylint is ued to unify the style of coding. the reason why it is unified is because when Developers write code, it is readable and understandable by other deveoplers that may need to edit/change the code.

What is Object Oriented programming (OOP)?

OOP is a computer programming model that organizes software design around data, or objects, rather than functions and logic. An object can be defined as a data field that has unique attributes and behavior. So rather than using If-statements, for loops, in project, OOP is used to eliminate the usage of them by working around data, and objects.
There are a couple of techniques that are used in OOP such as:
Factory a function or method that returns objects of a varying class. Elimantes using the same object
Instantiation the creation of a real instance or particular realization of an abstraction or template such as a class of objects
Class a blueprint for creating objects. Integers are built in objects, however we can create our own objects by creating classes
Namespace a collection of currently defined symbolic names along with information about the object that each name references. You can think of a namespace as a dictionary in which the keys are the object names and the values are the objects themselves.
Constructor a special kind of method that Python calls when it instantiates an object using the definitions in your class
Fixture define the steps and data that constitute the arrange phase of a test
Type Hint formal solution to statically indicate the type of a value within your Python code
Type Cast the method to convert the variable data type into a certain data type
Unit Test a software testing method by which individual units of source code are put under various tests to determine whether they are fit for use
Class Method method which is bound to the class and not the object of the class
Facade a structural design pattern that provides a simplified (but limited) interface to a complex system of classes
Design Patterns Used we use Factory designs, Fixture designs, and many others
Method a function that is available for a given object because of the object's type
Property object can be either mutable or immutable based on the type of data they hold
